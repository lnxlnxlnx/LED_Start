# 07. C 底层写法与内存模型

这一份专门补“底层感觉”：指针、内存布局、回调、字节操作这些到底怎么连起来。

## 1. 先抓核心心智模型

在 XinYueC 里，容器本质是这四件东西：

1. 一块连续内存 `m_data`
2. 每个元素大小 `m_typeSize`
3. 元素数量 `m_size`
4. 容量 `m_capacity`

你看到的所有操作，最终都在做“地址计算 + 字节拷贝/移动”。

## 2. 最关键的一行地址公式

```c
element_ptr = (char*)m_data + index * m_typeSize;
```

为什么先转 `char*`：因为 C 里只有 `char*` 的步进是按字节。  
这是泛型容器的基本功。

## 3. `void*` 不是“万能类型安全”

`void*` 只是“无类型指针”，不是类型检查器。  
这意味着你必须自己保证三件事：

- 传入类型和 `typeSize` 一致
- 强转时类型匹配
- 生命周期不悬空

如果这三件事没守住，编译器大概率帮不了你。

## 4. `memcpy` 的边界

很多新手会把 `memcpy` 当万能方案，但它只适合：

- POD 值类型
- 没有内部资源所有权的结构体

一旦结构体有 `char*`、句柄、文件描述符等成员，就要引入自定义：

- copy 回调
- deinit 回调

这也是 `XContainerSetDataCopyMethod / XContainerSetDataDeinitMethod` 存在的原因。

## 5. 扩容时要关注什么

扩容核心步骤通常是：

1. 计算新容量
2. 申请新内存或 `realloc`
3. 迁移旧数据
4. 更新 `m_data/m_capacity`
5. 失败回滚或保持旧状态

你读 `XVector_virtual.c` 时重点看：

- 何时扩容（满了才扩）
- 扩容倍数策略（小容量翻倍，大容量放缓）
- 失败时是否会破坏对象状态

## 6. 迭代器在 C 里本质是什么

XinYueC 的 `XVector_iterator` 本质只有一个字段：

```c
typedef struct XVector_iterator
{
    void* data;
} XVector_iterator;
```

它不是语法级对象，只是“当前位置指针”包装。  
`for_each_iterator(...)` 只是宏展开成 `for` 循环。

## 7. 函数指针回调是“运行时策略”

比如排序时：

- 比较逻辑由 `XCompare` 决定
- 升降序由 `XSortOrder` 决定

这就是运行时策略注入。好处是同一套算法可复用到不同类型。

## 8. 三类最常见底层 Bug

### Bug 1：越界访问

- `index` 未校验
- `size` 和 `capacity` 更新顺序错误

### Bug 2：双重释放或泄漏

- 元素有堆内存但没设 `deinit`
- move 后源对象没清空

### Bug 3：比较函数不一致

- 排序、查找、优先队列用的 compare 规则不一致

在你这个工程里，优先先防第 2 和第 3 类。

## 9. 建议的自查清单

每次写底层容器代码，至少问自己 8 个问题：

1. `NULL` 入参都处理了吗
2. `typeSize==0` 拦住了吗
3. 扩容失败对象还能继续用吗
4. `size/capacity` 会不会失真
5. 是否存在悬空指针
6. copy/move/deinit 是否成套
7. compare 是否在使用前设置
8. delete 是否会重复释放

把这张清单当成你的“底层代码验收标准”。

## 10. 你现在最该做的练习

做一个 `XVector<Student>`，其中 `Student` 含 `char* name`，并完整实现：

- `student_copy`
- `student_move`
- `student_deinit`
- `student_compare`

这个练习做完，你的底层能力会明显提升，因为它覆盖了内存与所有权的核心难点。
