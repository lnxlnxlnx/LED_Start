# 01. 快速上手

这一份只解决一个目标：先把 XinYueC 用起来。

## 1. 当前工程已经接入了什么

你现在这个仓库里，XinYueC 已经完成了基本接入：

- `XinYueC/*.c` 已经加入工程
- `XinYueC/include/` 已经加入头文件搜索路径
- `USER/main.c` 里已经调用了测试函数

所以你现在不需要重新移植，直接写测试代码即可。

## 2. 最常见的第一个例子：XVector 存 `int`

```c
#include "XVector.h"
#include "XCompare.h"

void demo_vector_int(void)
{
    XVector* vec = XVector_Create(int);
    XContainerSetCompare(vec, (XCompare)XCompare_int);

    int a = 10;
    int b = 20;
    int c = 30;

    XVector_push_back_base(vec, &a);
    XVector_push_back_base(vec, &b);
    XVector_push_back_base(vec, &c);

    for (size_t i = 0; i < XVector_size_base(vec); ++i)
    {
        int* p = (int*)XVector_at_base(vec, i);
        printf("vec[%d] = %d\r\n", (int)i, *p);
    }

    XVector_sort_base(vec, XSORT_ASC);
    XVector_delete_base(vec);
}
```

## 3. 为什么这里要先 `XContainerSetCompare`

这是当前版本里最容易忽略的一点。

- `XQuickSort()` 这种“直接排数组”的接口，会把比较函数作为参数传进去
- `XVector_sort_base()` 这种“排容器”的接口，会从容器内部取 `m_compare`

也就是说，`XVector_sort_base(vec, ...)` 想正常工作，前提通常是：

```c
XContainerSetCompare(vec, (XCompare)XCompare_int);
```

如果你没设置比较函数，`sort` 这类接口就不安全。

## 4. 原生数组排序

```c
#include "XSort.h"

void demo_array_sort(void)
{
    int arr[] = {9, 5, 1, 7, 3, 8, 2, 4, 6, 0};
    int n = sizeof(arr) / sizeof(arr[0]);

    XQuickSort(arr, n, sizeof(int), (XCompare)XCompare_int, XSORT_ASC);
}
```

这种方式最直接，适合先验证排序算法本身。

## 5. 自定义结构体怎么放进容器

先定义类型和比较函数：

```c
typedef struct
{
    int id;
    int score;
} Student;

int32_t Student_compare(const void* lhs, const void* rhs)
{
    const Student* a = (const Student*)lhs;
    const Student* b = (const Student*)rhs;

    if (a->score < b->score) return XCompare_Less;
    if (a->score > b->score) return XCompare_Greater;
    return XCompare_Equality;
}
```

使用时：

```c
XVector* vec = XVector_Create(Student);
XContainerSetCompare(vec, Student_compare);
```

如果结构体里只有值类型成员，到这里就够了。

## 6. 如果结构体里还有指针成员

例如：

```c
typedef struct
{
    char* name;
    int score;
} Student;
```

这时要继续考虑：

- 拷贝时是否要深拷贝
- 删除元素时是否要释放 `name`

也就是要给容器设置：

- `XContainerSetDataCopyMethod`
- `XContainerSetDataMoveMethod`
- `XContainerSetDataDeinitMethod`

否则容器默认只是按字节拷贝，很容易产生悬空指针或重复释放问题。

## 7. 生命周期要记住这三个词

- `init`：初始化一个已经存在的对象
- `deinit_base`：释放对象内部资源，但不释放对象本身
- `delete_base`：先析构，再释放对象本身

如果你是 `XVector_create()` 创建出来的，一般配套用 `XVector_delete_base()`。
