# 13. 思考题与实战训练

这份文档用于把你前面学的内容“逼成可执行能力”。  
建议做题方式：每道题都在 `IO/XinYue/` 新建一个 `test_xxx.h`，再在 `USER/main.c` 临时调用验证。

## 使用规则

1. 每题都先写你自己的“预期行为”，再写代码。
2. 每题都至少给一个失败用例（边界输入）。
3. 每题做完写 3 句话复盘：哪里卡住、怎么排查、下次怎么避免。

---

## A. 基础题（面向 01~05）

### A1. Vector 基本操作闭环

题目：

- 用 `XVector<int>` 实现 `push_back -> at -> pop_back -> delete` 完整流程。
- 输入序列：`{3, 1, 4, 1, 5}`。

验证标准：

- 遍历输出顺序正确。
- `pop_back` 后大小减少 1。
- 不出现异常重启或死循环。

思考题：

- `XVector_create` 和 `XVector_init` 在生命周期上有什么本质区别？

### A2. 原生数组排序 vs 容器排序

题目：

- 用 `XQuickSort` 排 `int arr[]`。
- 用 `XVector_sort_base` 排 `XVector<int>`。

验证标准：

- 两者最终升序一致。
- 容器排序前明确设置 `XContainerSetCompare`。

思考题：

- 为什么数组排序可以直接传 compare，而容器排序通常要先设置到对象里？

### A3. 自定义结构体排序

题目：

- 定义 `Student{id, score}`。
- 写 `Student_compare`，按 `score` 升序排序。

验证标准：

- 排序后 `score` 单调不降。
- 相同 `score` 时顺序行为符合你定义。

思考题：

- 你的 compare 是否满足“自反、对称、传递”？

## B. 进阶题（面向 06~09）

### B1. 看懂一次虚表派发

题目：

- 追踪一次 `XVector_push_back_base` 的调用链，写出 5 步流程（函数名级别）。

验证标准：

- 流程里包含：`class_init`、`enum 索引`、`XClassGetVirtualFunc`。

思考题：

- 如果 `table[]` 顺序和 `EXVector_...` 顺序错位，会出现哪类 bug？

### B2. 手写一个最小 OOP 示例

题目：

- 按 `08` 文档写一个最小 `Base + Derived + vtable` 示例。
- 实现 `print` 虚函数，派发两种对象。

验证标准：

- 同一 `Base*` 接口能调用到不同实现。

思考题：

- 你这个示例里，哪一行最能体现“多态是运行时绑定”？

### B3. 继承宏理解题

题目：

- 找 `XContainerObject_class_init`、`XVector_class_init`、`XQueue_class_init`。
- 解释它们为什么都要先 `XVTABLE_INHERIT_DEFAULT(...)`。

验证标准：

- 你的解释里必须出现“父类槽位复制”这个点。

思考题：

- 如果省略 `XVTABLE_INHERIT_DEFAULT`，最先出问题的通常是哪个层级的函数调用？

## C. 专题题（面向 10~12）

### C1. 迭代器失效实验

题目：

- 创建 `XVector<int>`，先拿 `it=begin`。
- 中途执行可能触发扩容的 `push_back`，再继续用旧 `it` 访问。

验证标准：

- 记录现象并解释原因（旧迭代器失效）。

思考题：

- 在你自己的业务代码里，哪些操作后必须重建迭代器？

### C2. Memory 后端替换实验

题目：

- 包装一组 `my_malloc/my_free`（可统计次数）。
- 用 `XMemory_setMethod` 注入。
- 跑 `XVector` 压入/删除测试。

验证标准：

- 能打印分配与释放计数。
- 程序行为与未替换前一致。

思考题：

- 为什么“全库统一通过 XMemory 入口分配”比散落 `malloc/free` 更可维护？

### C3. Compare 健壮性实验

题目：

- 给 `Student` 写两个 compare：按 `score`、按 `id`。
- 分别排序并比较结果。

验证标准：

- 两种排序结果确实体现规则差异。
- compare 返回值只使用 `Less/Equal/Greater` 三种。

思考题：

- 如果 compare 不满足传递性，排序算法可能表现出什么现象？

## D. 源码阅读题（拉高上限）

### D1. 阅读 `XVector_virtual.c`

问题：

1. 扩容策略是什么（小容量与大容量）？
2. `clear` 为什么只改 `size`，而不总是释放底层内存？
3. 哪些函数依赖 `XContainerDataDeinitMethod`？

### D2. 阅读 `XVtable.c`

问题：

1. `XVtable_append_vtable` 与 `XVtable_append_array` 的关系是什么？
2. 为什么“追加”本质上是顺序拼接？
3. 何时会触发容量扩展？

### D3. 阅读 `XQueue.c` 与 `XPriorityQueue_virtual.c`

问题：

1. `XQueue` 为什么更像“组合复用链表”而不是重写整套容器？
2. `XPriorityQueue` 如何依赖 compare 和 order 决定优先级？

## E. 自我评估表（每周一次）

按 0~2 分打分（0=不会，1=会用，2=能讲清原理）：

1. 我能解释 `XVector` 的生命周期管理。
2. 我能写正确的自定义 compare。
3. 我知道哪些操作会让迭代器失效。
4. 我能说清 `XVTABLE_INHERIT_DEFAULT` 的作用。
5. 我能追踪一次虚表派发链路。
6. 我能替换 `XMemory` 后端并验证行为。

总分建议：

- `0~4`：先回到 `01~05`
- `5~8`：继续 `06~09`
- `9~12`：重点做 `10~13` 里的源码题和实验题
