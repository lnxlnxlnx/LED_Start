# 10. 迭代器机制与实践

这一份专门讲 XinYueC 的 iterator。目标是让你知道它“本质是什么、怎么安全用、哪里容易踩坑”。

## 1. 先看本质

XinYueC 的迭代器不是 C++ 那种语言内建对象，本质是“当前位置指针的轻量封装”。

以 `XVector_iterator` 为例：

```c
typedef struct XVector_iterator
{
    void* data;
} XVector_iterator;
```

所以它能做的事情很朴素：

- `begin`：指向第一个元素
- `end`：用 `NULL` 作为结束哨兵
- `add`：步进到下一个元素
- `data`：取当前位置元素地址

## 2. 正向迭代器常用接口

- `XVector_begin`
- `XVector_end`
- `XVector_iterator_add`
- `XVector_iterator_equality`
- `XVector_iterator_data`

典型手写遍历：

```c
XVector_iterator it = XVector_begin(vec);
XVector_iterator end = XVector_end(vec);

while (!XVector_iterator_equality(&it, &end))
{
    int* v = (int*)XVector_iterator_data(&it);
    printf("%d\r\n", *v);
    XVector_iterator_add(vec, &it);
}
```

## 3. `for_each_iterator` 宏怎么用

`XContainerObject_iterator.h` 里提供了统一遍历宏：

```c
for_each_iterator(container, XVector, it)
{
    int* v = (int*)XVector_iterator_data(&it);
    ...
}
```

这会展开成普通 `for`。优点是写法统一，缺点是出错时需要理解宏展开。

## 4. 反向迭代器

`XVector_reverse_iterator` 与正向类似，关键接口：

- `XVector_rbegin`
- `XVector_rend`
- `XVector_reverse_iterator_add`
- `XVector_reverse_iterator_data`

适合倒序遍历或逆向处理场景。

## 5. 迭代器失效规则（最重要）

在容器发生结构变化后，旧迭代器可能失效。尤其是 `XVector`：

- `push_back/insert/resize` 触发扩容后，原数据区地址可能变化
- 原迭代器里的 `data` 指针仍指向旧地址

因此规则是：

1. 修改容器结构后，不要继续用旧迭代器
2. 重新 `begin/end` 或重新获取当前位置

## 6. 遍历中删除怎么做

推荐用库提供的 `erase` 语义（可返回下一个位置），不要边遍历边手动 `remove` 当前项且继续 `add` 旧迭代器。

安全策略：

1. 先保存或获取“下一个位置”
2. 删除当前
3. 使用新位置继续

## 7. 与 `XFor_each` 回调遍历的关系

`XVector_iterator_for_each` 这类接口最终也在做 iterator 循环，只是把每个元素交给回调函数。

适合：

- 遍历逻辑固定
- 想减少样板代码

不适合：

- 遍历中复杂修改容器结构

## 8. 实战建议

先做两个最小练习：

1. 正向迭代打印 `XVector<int>`
2. 反向迭代打印 `XVector<int>`

再做第三个：

3. 遍历时按条件删除元素，验证你对“迭代器失效”是否掌握
