# 08. C 实现面向对象实战

你可以把 XinYueC 理解为：在 C 里手工实现了“对象 + 继承 + 多态”的一套简化框架。

## 1. C 里做 OOP 的三件套

1. 对象：结构体承载状态
2. 继承：首成员嵌入父类结构
3. 多态：函数指针表（虚表）派发

XinYueC 对应关系非常明确：

- `XClass`：根类，持有 `m_vtable`
- `XContainerObject`：容器基类
- `XVector/XQueue/...`：派生类
- `XVtable`：虚函数表容器

## 2. 首成员继承为什么成立

例如：

```c
typedef struct XVector
{
    XContainerObject m_class;
} XVector;
```

因为首成员地址与结构体起始地址一致，`XVector*` 可安全转为 `XContainerObject*`。  
这就是 C 里最常见的“继承”技巧。

## 3. 虚表派发怎么理解

调用流程可以简化成：

1. 对象里拿到 `m_vtable`
2. 按枚举索引取函数指针
3. 强转为目标函数类型
4. 执行

这和 C++ 虚函数的思想一致，只是这里完全手写。

## 4. 用最小例子自己写一个 OOP 风格模块

下面给你一个极简模板（可单独练习）：

```c
typedef struct Base Base;
typedef struct Derived Derived;

typedef struct
{
    void (*deinit)(Base* self);
    void (*print)(Base* self);
} BaseVTable;

struct Base
{
    const BaseVTable* vptr;
};

struct Derived
{
    Base base;   // 首成员继承
    int value;
};
```

```c
static void Derived_deinit(Base* self)
{
    // cleanup
    (void)self;
}

static void Derived_print(Base* self)
{
    Derived* d = (Derived*)self;
    printf("value=%d\r\n", d->value);
}

static const BaseVTable g_derived_vtbl = {
    Derived_deinit,
    Derived_print
};
```

```c
void Derived_init(Derived* d, int value)
{
    d->base.vptr = &g_derived_vtbl;
    d->value = value;
}

void Base_print(Base* obj)
{
    obj->vptr->print(obj);
}
```

这个模板就是 XinYueC 思路的最小版。

## 5. XinYueC 里 OOP 的实际价值

不是为了“像 C++ 一样炫”，而是为了三件实用目标：

- 同一套接口适配不同容器实现
- 让 copy/move/deinit 规则可替换
- 保持嵌入式项目里的模块边界

## 6. 和 C++ OOP 的差异

你要清楚这些差异，避免误解：

- 没有编译器自动类型检查虚调用
- 没有自动析构链，顺序要自己维护
- 没有语言级 `private/protected`
- 宏和强转用得多，调试门槛更高

所以 C 版 OOP 更像“工程手册 + 纪律”，不是语言保障。

## 7. 写 C 风格 OOP 的实践规则

### 规则 1：对象状态必须可恢复

`init` 失败后对象要么可重试，要么明确不可用，不能半初始化。

### 规则 2：析构路径必须单一

统一 `deinit` 做内部释放，`delete` 只负责 `deinit + free`。

### 规则 3：对外接口不要暴露实现细节

调用方尽量看不到内部字段，避免把抽象层打穿。

### 规则 4：虚表只做行为分发，不存业务状态

状态应该在对象里，虚表只存函数。

## 8. 你在当前仓库可以做的两个实战

1. 按上面最小模板，自己写一个 `XDevice` 抽象：`open/read/write/close`
2. 写两个派生实现：一个“串口模拟设备”，一个“内存缓冲设备”，都走同一套基类接口

这两个练习是从“看懂 XinYueC”到“你自己能设计框架”的关键跨越。

## 9. 学习建议

你现在不要一次把所有宏都吃透。建议节奏：

1. 先把“首成员继承 + 虚表派发”手写一遍
2. 再回读 `XClass.h / XVtable.h`
3. 最后看 `XVector_class_init()` 里的虚表继承和重载

这样理解速度会比直接啃全量源码快很多。
