## ✨ 2. 快速开始（最小步骤）

### 步骤 1：包含头文件

```c
#include "lw_oopc.h"
```

### 步骤 2：定义接口和类

```c
INTERFACE(IMoveable)
{
    void (*move)(IMoveable* t);
};

ABS_CLASS(Animal)
{
    char name[32];
    void (*say_hello)(Animal* t);
};

CLASS(Dog)
{
    EXTENDS(Animal);
    IMPLEMENTS(IMoveable);
};
```

### 步骤 3：实现构造/析构和行为绑定

```c
static void Dog_move(IMoveable* t)
{
    (void)t;
    printf("dog run\r\n");
}

CTOR(Dog)
SUPER_CTOR(Animal);
FUNCTION_SETTING(IMoveable.move, Dog_move);
END_CTOR

DTOR(Dog)
return Animal_dtor(SUPER_PTR(cthis, Animal));
END_DTOR
```

### 步骤 4：创建对象并通过接口调用

```c
Dog* dog = Dog_new(lw_oopc_file_line);
IMoveable* m = SUPER_PTR(dog, IMoveable);
m->move(m);
Dog_delete(dog);
```

## ✅ 4. 你自己写业务对象的推荐套路

### 4.1 设计阶段

1. 先定接口（外部只关心什么行为）
2. 再定抽象基类（共享状态和共享逻辑）
3. 最后定具体类（差异行为）

### 4.2 编码阶段

1. 先写声明宏：`INTERFACE/ABS_CLASS/CLASS`
2. 再写 `ctor`，完成函数指针绑定
3. 再写 `dtor/finalize`，闭合资源释放路径
4. 最后写测试函数，从 `main` 临时调用验证

### 4.3 回收阶段

统一原则：谁 `new`，谁负责 `delete`，不要跨层级“偷释放”。

## ⚠️ 5. 常见错误与排查

### 错误 1：直接 `free(obj)`

现象：资源释放不完整、析构链断掉。  
正确做法：必须 `Type_delete(obj)`。

### 错误 2：忘记 `SUPER_CTOR(Father)`

现象：父类函数指针未初始化，调用崩溃。  
正确做法：子类 `CTOR` 里先调用 `SUPER_CTOR`。

### 错误 3：`SUB_PTR` 类型不匹配

现象：地址偏移错误，行为异常。  
正确做法：只在你明确真实对象类型时使用 `SUB_PTR`。

### 错误 4：引用计数对象未对称释放

现象：对象泄漏或提前释放。  
正确做法：每次持有引用时递增，对应路径上调用 `delete` 递减。
