## 2. 知识点讲解（初学者版）

## 2.1 什么是命令模式

命令模式把“要做什么”封装成一个对象（或结构体）。

在 C 里常见表示：

```c
typedef struct {
    void (*execute)(void* ctx);
    void* ctx;
} Command;
```

这样你可以先把命令放进队列，之后再统一执行。

## 2.2 为什么嵌入式里好用

典型价值：

- 中断里只入队，不做重活
- 主循环按节奏执行命令
- 更容易做重放、限流和优先级扩展

## 2.3 三问法：什么时候用、映射到哪里、第一步做什么

### 什么时候用

当“触发时机”和“执行时机”不一致，或者执行逻辑很多时使用。

### 在本项目映射到哪里

- 事件来源：`HARDWARE/EXTI/exti.c`
- 执行逻辑：`IO/key_led_one.c`
- 容器支撑：`XinYueC/include/XQueue.h`

### 第一最小实现步

把按键中断事件改成“入队命令”，主循环里“出队执行”。

## 3. 本仓库动手步骤

目标：构建按键命令队列，不再依赖单个全局事件变量。

### Step 1：定义命令结构

在 `IO/` 新建：

- `command_queue.h`
- `command_queue.c`

定义命令类型（如 `CMD_LED_BLINK`、`CMD_SPEED_UP`、`CMD_SPEED_DOWN`）。

### Step 2：建立队列

使用 `XQueue`（或 `XCircularQueue`）保存命令对象。

### Step 3：中断只入队

在 `exti.c` 的中断里，只做轻量操作：构造命令并入队。

### Step 4：主循环统一执行

在 `key_led_one_loop` 或主循环中出队并分发执行。

### Step 5：加入日志

给“入队/出队/执行失败”打日志，确认系统行为可观察。

## 4. 常见坑

1. 在中断里做复杂逻辑，导致中断占用过长。
2. 队列满了没有处理策略（丢弃/覆盖/阻塞）。
3. 命令上下文指针指向了已失效内存。
