## 2. 知识点讲解（初学者版）

## 2.1 为什么要学这节

设计模式在 C 里能不能写稳，底层关键是：

- 指针是否安全
- 内存生命周期是否清晰
- 资源是否能正确释放

这节就是打这些基础。

## 2.2 `void* + typeSize` 是什么

XinYueC 容器并不知道你放的是 `int` 还是 `struct`。  
所以它用：

- `void*` 指向原始内存
- `typeSize` 表示单个元素大小

访问第 `i` 个元素时，本质是：

```c
(char*)data + i * typeSize
```

## 2.3 `copy / move / deinit` 三件事

### copy

- 含义：复制一份数据到容器
- 适用：值类型或可深拷贝对象

### move

- 含义：转移资源所有权，减少复制开销
- 适用：大对象或内部含动态资源

### deinit

- 含义：元素删除或容器销毁时执行释放逻辑
- 适用：元素内部有 `malloc` 资源

## 2.4 为什么很多 bug 都在这里

常见场景：

- 结构体里有 `char*`，但你只 `memcpy`，最后重复释放或泄漏。
- 容器删元素时没有 deinit，资源一直不释放。

## 3. 本仓库动手步骤

目标：实现 `XVector<Student>`，并正确排序与释放。

### Step 1：定义结构体（先做简单版）

```c
typedef struct
{
    int id;
    int score;
} Student;
```

### Step 2：定义比较函数

```c
int32_t Student_compare(const void* a, const void* b)
{
    const Student* lhs = (const Student*)a;
    const Student* rhs = (const Student*)b;
    if (lhs->score < rhs->score) return XCompare_Less;
    if (lhs->score > rhs->score) return XCompare_Greater;
    return XCompare_Equality;
}
```

### Step 3：创建并设置 compare

```c
XVector* vec = XVector_Create(Student);
XContainerSetCompare(vec, Student_compare);
```

### Step 4：压入、排序、遍历

- `XVector_push_back_base`
- `XVector_sort_base`
- `XVector_at_base`

### Step 5：销毁

- `XVector_delete_base(vec)`

## 4. 进阶练习（含动态内存）

把结构体改成：

```c
typedef struct
{
    char* name;
    int score;
} Student2;
```

然后补三件事：

1. `copy`：深拷贝 `name`
2. `move`：转移 `name` 指针并清空源
3. `deinit`：释放 `name`

## 5. 常见坑

1. 忘记 `XContainerSetCompare`，导致容器排序行为不可靠。
2. 使用 move 语义后还继续使用源对象资源。
3. `deinit` 里释放了不属于该元素的内存。
