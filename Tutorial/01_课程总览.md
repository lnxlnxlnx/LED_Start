# 01. 课程总览

## 学习目标

完成课程后，你应具备这 4 项能力：

1. 能解释并实现嵌入式场景下常用设计模式（不是背定义）。
2. 能把 XinYueC 中的回调、虚表、容器机制映射到模式思想。
3. 能把“模块抽象 + 模式”用于重构你现有工程。
4. 能独立设计一套可扩展的驱动/业务框架（支持后续传感器和功能迭代）。

## 课程结构

- 阶段 A（第 1-3 周）：基础打底，建立统一工程视角
- 阶段 B（第 4-7 周）：核心模式，和 XinYueC 的机制一一对应
- 阶段 C（第 8-10 周）：工程化模式组合，面向实际模块重构
- 阶段 D（第 11-12 周）：项目实战与验收

## 前置要求

- 能读 C 指针、结构体、函数指针
- 能独立编译并下载当前工程
- 能在 `IO/`、`HARDWARE/`、`USER/` 增加测试代码

## 学习原则

1. 先“会用”再“抽象”：先跑通，再提炼模式。
2. 一个模式至少做一个真实改造，不只看例子。
3. 每周只引入 1-2 个新模式，避免概念堆积。
4. 所有模式都回答同一问题：它解决了当前项目的什么痛点。

## 每周标准动作

1. 阅读本周主题（1-2 小时）
2. 在仓库里实现最小例子（2-4 小时）
3. 改造一个真实模块（2-6 小时）
4. 写复盘（15 分钟）
