# 多功能秒表系统 — 设计与实现

## 一、考试要求

基于正点原子 NANO STM32F103 开发板，实现数码管多功能秒表：

| 功能 | 分值 | 说明 |
|------|------|------|
| 数码管动态扫描 | 15分 | 8位数码管无闪烁/重影 |
| 高精度计时(10ms中断) | 15分 | 厘秒精度，范围 00-00-00 ~ 99-59-99 |
| KEY1 启动/暂停 | 15分 | 带按键消抖 |
| KEY2 暂停清零(防误触) | 10分 | 运行中按下无效 |
| 记次冻结/3秒自动恢复 | 15+15分 | 冻结显示，后台继续计时 |
| LED 1Hz交替(DS0/DS1) | 2分 | 随秒变化 |
| 流水灯(DS2~DS7) | 3分 | 始终运行，不受暂停/冻结影响 |
| 串口日志 (115200) | 10分 | 中文状态输出 |

## 二、架构总览

```
┌──────────────────────────────────────────────────────────┐
│                    中断层 (1ms)                           │
│  TIM3_IRQHandler() at 1ms                                │
│  ├── g_tim3.tick++          ← 全局 tick 时间基准          │
│  ├── LED_SMG_Scan()         ← 数码管动态扫描 (交替消影)    │
│  └── Stopwatch_TimerTick()  ← 秒表逻辑                    │
│       ├── sw_ReadKeys()     ← 按键采样 + 20ms消抖         │
│       └── (每 g_tick_10ms)                                │
│            ├── sw_UpdateTime()     ← 厘秒累加+进位         │
│            │    └── sw_UpdateDisplay()  ← 刷新数码管      │
│            ├── 冻结倒计时            ← 3秒自动恢复         │
│            └── sw_UpdateLEDs()       ← DS0~DS7 控制      │
│                 ├── DS0/DS1 1Hz 交替                      │
│                 └── DS2~DS7 流水灯                        │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│                    主循环 (≈50Hz)                         │
│  while(1) { delay_ms(20); }  ← 仅空转，不处理其他事务    │
└──────────────────────────────────────────────────────────┘
```

### 设计原则

- **所有实时工作在 ISR 中完成**：按键消抖、厘秒累加、LED 控制、数码管刷新都在 TIM3 中断里处理。
- **主循环不负责任务逻辑**：只做 `delay_ms(20)` 保持 CPU 空闲，方便将来扩展。
- **时间参数运行时计算**：通过 `TIMER_MS(&g_tim3, ms)` 宏自动适配 TIM3 的 arr/psc，改变定时器配置无需修改代码。

## 三、状态机

```
        ┌─────────────────────────────────────────────┐
        │                                             │
        ▼                                             │
  ┌──────────┐   KEY1    ┌──────────┐   KEY1    ┌─────┴──────┐
  │ STOPPED  │ ────────→ │ RUNNING  │ ────────→ │  PAUSED    │
  │ (归零)   │ ←──────── │          │ ←──────── │            │
  └──────────┘   KEY1    └────┬─────┘   KEY1    └─────┬──────┘
       ▲                      │                       │
       │                   WK_UP                     KEY2
       │                      │                       │
       │                      ▼                       ▼
       │               ┌──────────────┐          STOPPED
       │               │ LAP_FROZEN   │          (清零)
       │               │ (冻结3秒)    │
       │               └──────┬───────┘
       │                      │
       │               ┌──────┴───────┐
       │               │  3秒后自动   │
       │               │  → RUNNING   │
       │               │  KEY1 → PAUSED
       └───────────────┴──────────────┘
```

### 状态说明

| 状态 | 定义 | 厘秒累加 | 数码管 | DS0/DS1 |
|------|------|---------|--------|---------|
| `STOPPED` | 停止/清零 | ✗ | 显示 00-00-00 | 灭 |
| `RUNNING` | 运行中 | ✓ | 实时更新 | 1Hz 交替 |
| `PAUSED` | 已暂停 | ✗ | 定格当前值 | 灭 |
| `LAP_FROZEN` | 记次冻结 | ✓(后台) | 显示冻结值 | 1Hz 交替 |

## 四、按键映射

| 按键 | 引脚 | 电平逻辑 | 功能 |
|------|------|----------|------|
| **KEY1** | PC9 | 按下=0 (上拉) | 运行↔暂停、停止→启动、冻结→暂停 |
| **KEY2** | PD2 | 按下=0 (上拉) | 暂停/停止时清零（防误触，运行中无效） |
| **WK_UP** | PA0 | 按下=1 (下拉) | 运行中记次冻结 |

### 消抖机制（软件消抖）

```
 ┌─ 原始电平采样 ─────────────────────────────┐
 │  每 1ms 读一次 GPIO，统一转为 1=按下         │
 │  连续 g_key_debounce 个 tick 电平不变则确认  │
 └────────────────────────────────────────────┘
 
 KEY1(PC9):  raw[K_IDX_KEY1] = !PCin(9)
 KEY2(PD2):  raw[K_IDX_KEY2] = !PDin(2)
 WK_UP(PA0): raw[K_IDX_WKUP] =  PAin(0)
```

- 默认消抖时间 20ms（即连续 20 个 tick 电平一致）
- 消抖时间随 `g_tim3.tick_us` 自动缩放
- 检测到**上升沿**（松开→按下）时才触发一次动作，不会连发

## 五、时间格式与进位

### 数码管布局（8位）

```
 位0  位1   位2   位3  位4   位5   位6  位7
[M十][M个] [ - ] [S十][S个] [ - ] [C十][C个]
```

段码：横线 `-` = `0x02`（共阴，仅 g 段亮）

### 进位链

```
厘秒(10ms)  +1
   │  ≥ 100
   ▼
秒          +1
   │  ≥ 60
   ▼
分          +1
   │  > 99
   ▼
归零 → STOPPED
```

- 最大显示 `99-59-99`（99分59秒99厘秒）
- 溢出后自动归零并状态切回 STOPPED

## 六、LED 控制

### DS0/DS1 — 1Hz 交替闪烁（对应秒数）

```c
if (g_time.second & 0x01)
    { LED0=灭, LED1=亮 }    // 奇数秒
else
    { LED0=亮, LED1=灭 }    // 偶数秒
```

- 仅 `RUNNING` 和 `LAP_FROZEN` 状态点亮
- `PAUSED` / `STOPPED` 状态全灭

### DS2~DS7 — 流水灯（始终运行）

```
方向: DS2 → DS3 → DS4 → DS5 → DS6 → DS7 → DS2 → ...
间隔: 500ms (由 g_flow_interval 控制，自动适配 tick)
特点: 不受暂停、停止、冻结状态影响，一旦上电就持续流动
```

## 八、API 参考

### stopwatch.h

```c
void Stopwatch_Init(void);
```
- **说明**：初始化秒表模块，清零状态和时间，LED 全灭，数码管显示 `00-00-00`。
- **调用时机**：在 `main()` 中，TIM3 启动后、`My_TIM3_Init()` 之后调用一次。
- **副作用**：用 `TIMER_MS(&g_tim3, ...)` 计算所有时间参数并存入静态变量。

```c
void Stopwatch_TimerTick(void);
```
- **说明**：秒表核心逻辑，每次 TIM3 中断调用一次（1ms）。
- **内部处理**：按键消抖 → 厘秒累加 → 冻结倒计时 → LED 控制。
- **调用位置**：`HARDWARE/TIMER/timer.c` 的 `TIM3_IRQHandler` 中。

```c
Stopwatch_State Stopwatch_GetState(void);
```
- **说明**：获取当前秒表状态。
- **返回**：`SW_STOPPED=0` / `SW_RUNNING=1` / `SW_PAUSED=2` / `SW_LAP_FROZEN=3`

```c
Stopwatch_Time Stopwatch_GetTime(void);
```
- **说明**：获取当前计时值（用于外部查询）。
- **返回结构**：`{ u16 minute; u8 second; u8 centisecond; }`

### stopwatch.h 内部类型

```c
typedef enum {
    SW_STOPPED    = 0,    // 停止
    SW_RUNNING    = 1,    // 运行中
    SW_PAUSED     = 2,    // 已暂停
    SW_LAP_FROZEN = 3     // 记次冻结（显示冻结，后台计时）
} Stopwatch_State;

typedef struct {
    u16 minute;         // 分 (0~99)
    u8  second;         // 秒 (0~59)
    u8  centisecond;    // 厘秒 (0~99)
} Stopwatch_Time;
```

### 适配层

秒表依赖 `g_tim3` 和 `TIMER_MS()` 宏（定义在 `timer.h`）：

```c
#define TIMER_MS(pt, ms)  ((u32)((ms) * 1000 / (pt)->tick_us))
```

其中 `tick_us` 在 `My_TIM3_Init()` 中自动计算：
```c
g_tim3.tick_us = (arr+1) * (psc+1) / 72;  // 单位: 微秒
```

## 十、调试指南

| 现象 | 检查点 |
|------|--------|
| 数码管不亮 | `LED_SMG_Init()` 是否在 `My_TIM3_Init()` 之前调用 |
| 数码管闪烁 | `LED_SMG_Scan()` 是否在 ISR 中每 1ms 调用 |
| 按键无反应 | `My_KEY_Init()` 是否被调用；IO 电平是否正常 |
| KEY2 清不了零 | 确认当前是 PAUSED/STOPPED 状态（DS0/DS1 灭） |
| WK_UP 记次无效 | 确认当前是 RUNNING 状态（DS0/DS1 闪烁） |
| 流水灯不转 | ISR 中 `Stopwatch_TimerTick()` 是否调用 |
| 串口无日志 | `uart_init(115200)` 波特率与串口工具是否一致 |
| 3秒冻结不恢复 | 确认 TIM3 的 arr/psc 对应到正确的冻结 tick 数 |
